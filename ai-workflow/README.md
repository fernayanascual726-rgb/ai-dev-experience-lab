# AI Workflow

本目录用于记录 AI 如何嵌入研发流程、团队协作和日常开发节奏。

重点关注工作流设计，而不是单点工具使用。

## 适合收录的内容

- AI 辅助需求拆解、开发、测试、Review、发布的流程设计
- 人与 AI 的分工方式
- 团队使用 AI Coding 的协作规范
- AI 工具在现有工程流程中的接入方式

## 推荐写法

每个工作流建议说明：

1. 流程目标
2. 适用团队或场景
3. 步骤设计
4. 人工检查点
5. 风险和边界
6. 实施后的反馈

## 命名建议

```text
ai-assisted-code-review-flow.md
issue-to-pr-workflow.md
test-generation-workflow.md
```

## 维护原则

- 工作流要能被执行，而不只是描述理念
- 明确哪些环节由 AI 处理，哪些环节必须人工判断
- 记录团队协作成本和实际阻力
