# ai-dev-experience-lab

这是一个面向 **AI Developer Experience（AI 开发者体验）** 的长期知识库，用于沉淀 AI 在研发流程、编码协作、工作流设计和团队实践中的真实经验。

这里关注的不是概念本身，而是 AI 如何进入日常开发：在哪些场景有效，如何设计提示词，怎样嵌入工作流，以及它对开发者效率和协作体验带来的具体变化。

## 目标

- 记录 AI Coding 与 AI 工作流的可复用实践
- 总结开发者体验中的问题、约束和改进方法
- 沉淀适合长期维护的提示词、模板和案例
- 为技术分享、团队协作和工程落地提供素材

## 内容结构

| 目录 | 说明 |
| --- | --- |
| `ai-scene-case/` | AI 在具体研发场景中的使用案例 |
| `dev-prompt/` | 面向开发任务的提示词模板与写法 |
| `ai-workflow/` | AI 辅助研发流程与协作工作流 |
| `experience-summary/` | 开发者体验、效率和协作方面的总结 |
| `tech-sharing/` | 技术分享、内部交流和材料草稿 |

## 写作原则

- 以真实开发场景为起点
- 优先记录可复用的方法，而不是一次性结论
- 保留背景、约束和失败经验
- 避免空泛判断，尽量说明适用条件
- Markdown 风格保持简洁、清晰、可长期维护

## 建议内容格式

每篇内容建议包含：

1. 背景
2. 问题或目标
3. 使用方式
4. 过程记录
5. 结果与限制
6. 可复用经验

## 维护方式

这个仓库适合作为持续迭代的知识库。内容可以先以草稿形式记录，再逐步整理为案例、模板或分享材料。重点是保留实践上下文，让未来的复盘和复用更容易。
